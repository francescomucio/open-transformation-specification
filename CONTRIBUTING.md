# Contributing to the Open Transformation Specification

We welcome contributions to the Open Transformation Specification! This document provides guidelines for contributing to the project.

## How to Contribute

We welcome new contributors to the project whether you have changes to suggest, problems to report, or some feedback for us. Please jump to the most relevant section from the list below:

- Ask a question or offer feedback: use a discussion
- Suggest a change or report a problem: open an issue
- Contribute a change to the repository: open a pull request
- Or just get in touch

## Development Process

The Open Transformation Specification is currently in early development (v0.1.0). The development process is evolving as we establish the foundation for the specification.

### Current Focus Areas

- **Core Specification**: Defining the fundamental structure and concepts
- **Examples**: Creating clear examples of transformation specifications
- **Documentation**: Building comprehensive documentation
- **Community**: Establishing governance and contribution processes

## Specification Guidelines

When contributing to the specification:

- Use clear, unambiguous language
- Provide examples for complex concepts
- Consider backward compatibility
- Follow the established structure in `versions/v0.1.0/`

## Code Standards

- Use clear, descriptive commit messages
- Follow existing formatting conventions
- Include documentation for new features
- Test any code changes thoroughly

## Getting Help

- Check existing [issues](https://github.com/your-org/open-transformation-specification/issues)
- Start a [discussion](https://github.com/your-org/open-transformation-specification/discussions)
- Review the [Code of Conduct](CODE_OF_CONDUCT.md)
